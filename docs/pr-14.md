# PR 14：优化结构化纠错反馈

## 本 PR 做了什么

- 逐句反馈结果新增 `issues` 数组，用结构化方式描述本句最值得练习的问题。
- 每个问题包含：
  - 错误类型
  - 优先级
  - 问题片段
  - 推荐表达
  - 中文解释
  - 可直接跟读的练习句
- DeepSeek prompt 明确要求返回 `issues`，并限制错误类型和优先级枚举。
- 后端对真实模型返回的 `issues` 做归一化，缺失时从旧 `correction` 自动生成一条默认问题。
- mock 兜底同步返回结构化问题，保证未配置真实模型时 Demo 仍能展示完整纠错面板。
- 前端纠错卡改为优先展示结构化 issue 列表，并保留原来的总体改写和总体原因。
- 逐句反馈模式单独放宽 Qwen Realtime VAD 停顿阈值，避免用户一句话还没说完就被截断。
- 优化生成纠错后的左右布局：左侧输入区收敛为继续下一句，右侧结构化纠错更像报告卡片。
- 对话展示区新增“翻译”按钮，英文较弱的用户可以随时查看中文释义。

## 为什么这样做

题目要求关注“语法/表达纠错与纠错时机”。PR12 已经把纠错时机拆清楚：逐句反馈由用户主动点击后纠错，沉浸式对话结束后统一总结。

本 PR 进一步提升纠错精准度：不再只给一段泛泛说明，而是把问题拆成类型、优先级和可练习句，方便用户知道先改什么、怎么改、下一句怎么练。

## 核心实现思路

- `prompts.py` 在逐句反馈 prompt 中新增 `issues` JSON 结构要求。
- `ai_client.py` 新增 issue 类型和优先级归一化逻辑，保证前端收到稳定结构。
- `mock_feedback.py` 为面试、点餐、会议场景分别提供 mock issue。
- `App.jsx` 新增 `IssueCard`，展示错误类型、优先级、推荐表达和跟读练习句。
- `styles.css` 新增 issue 卡片、优先级标签和结构化纠错布局。
- `qwen_realtime.py` 按模式设置 VAD：逐句反馈使用更长静音窗口，沉浸式继续保持低延迟。
- `routes.py` 新增翻译接口，前端通过后端请求 DeepSeek 翻译，不暴露 API Key。
- `App.jsx` 为初始提示、用户消息和 AI 消息增加翻译按钮，并在气泡内展示译文。

## 如何测试

```bash
python -m compileall backend\app
npm run build
git diff --check
```

手动测试：

- 在逐句反馈模式说一句英文，等待 AI 接话后点击“获取完整纠错评分”。
- 确认右侧“纠错反馈”显示错误类型和优先级标签。
- 确认每条问题包含问题片段、推荐表达、中文解释和跟读练习句。
- 确认总体改写和总体原因仍然显示。
- 断开 DeepSeek Key 后，确认 mock 兜底仍能展示结构化纠错问题。
- 切换到沉浸式对话，确认对话中仍不展示即时纠错，只在结束后总结。
- 逐句反馈读一段较长英文，中间短暂停顿时不应立刻截断。
- 点击任意对话气泡里的“翻译”，确认中文译文显示在该气泡下方，再次点击可收起。
