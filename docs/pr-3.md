# PR 3: 浏览器录音采集

## 本 PR 做了什么

- 将前端“录音占位”按钮改为真实录音按钮。
- 使用浏览器原生 `navigator.mediaDevices.getUserMedia` 获取麦克风权限。
- 使用 `MediaRecorder` 采集音频并生成本地音频 URL。
- 页面展示录音时长、音频大小和回放控件。
- 保留文本框作为模拟语音识别结果输入。

## 为什么这样做

题目要求支持实时语音对话。当前阶段还不接入真实语音识别和大模型，所以 PR 3 先完成浏览器音频采集这一层，让后续 PR 可以基于真实录音继续接入语音识别或音频上传。

## 核心实现思路

- 点击“开始录音”时请求麦克风权限。
- 通过 `MediaRecorder` 收集音频片段。
- 点击“停止录音”后合成 `Blob`，生成可回放的本地 URL。
- 组件卸载时停止麦克风轨道并释放音频 URL，避免资源泄漏。

## 如何测试

启动前端：

```bash
npm run dev:frontend
```

打开：

```text
http://localhost:5173
```

测试步骤：

1. 点击“开始录音”。
2. 浏览器弹出权限提示时允许使用麦克风。
3. 说一句英文后点击“停止录音”。
4. 确认页面展示录音时长、音频大小和音频回放控件。
5. 在文本框输入模拟识别结果，点击“提交模拟语音”，确认 PR 2 的 mock 反馈仍正常。
