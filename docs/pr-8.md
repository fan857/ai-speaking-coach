# PR 8：支持重新朗读 AI 回复

## 本 PR 做了什么

- 在开场 AI 提示旁增加“重读”按钮。
- 在每条 AI 回复气泡旁增加“重读”按钮。
- 在右侧最新 AI 回复卡片中增加“重读”按钮。
- 点击“重读”后，使用浏览器 `SpeechSynthesis` 重新朗读对应英文内容。
- 保持原有多轮对话、语音识别、纠错反馈和评分流程不变。

## 为什么这样做

英语口语练习中，用户经常需要反复听 AI 的英文回复，模仿语音语调和表达节奏。PR6 已经实现了 AI 回复后自动朗读，但用户错过朗读或想重复听时，还缺少主动重播能力。

本 PR 只补充“重新朗读 AI 回复”这一项前端交互，不改后端和模型调用，符合“每个 PR 只做一件事”的要求。

## 核心实现思路

- 复用现有 `speakAiReply` 函数，不新增 TTS 服务。
- 新增 `handleReplayAiReply(text)`，接收指定 AI 文本并触发浏览器朗读。
- 对话区只给 AI 消息展示“重读”按钮，用户消息不展示。
- 右侧最新 AI 回复卡片复用同一个重读逻辑。
- 每次重读前会先 `speechSynthesis.cancel()`，避免多个朗读音频重叠。

## 如何测试

1. 启动前端：

```bash
npm run dev:frontend
```

2. 启动后端：

```bash
npm run dev:backend
```

3. 打开：

```text
http://localhost:5173
```

4. 输入一句英文并提交，确认 AI 回复会自动朗读。

5. 点击 AI 回复气泡旁的“重读”，确认会再次朗读该条英文回复。

6. 再输入第二句英文，确认多条 AI 回复各自都有“重读”按钮。

7. 点击右侧 AI 回复卡片的“重读”，确认会重新朗读最新 AI 回复。
