# PR 15：产品化页面与评审点展示优化

## 本 PR 做了什么

本 PR 不改后端逻辑和 API Key 相关代码，只优化现有 AI 英语口语陪练页面的信息架构与展示方式，让页面更像正式训练产品，并让评委能直接看到作品如何对应题目要求。

主要改动：

- 顶部新增产品 slogan、当前训练状态卡片和题目要求命中点标签。
- 左侧对话区改为“实时对话训练”，明确标注“语音识别结果”和“AI 语音回复”。
- 将“重读”改为“重听 AI 回复”，结束按钮改为“结束对话并生成课后报告”。
- 增加实时链路说明，强调识别、回复、朗读先行，详细纠错和总结延后生成。
- 右侧反馈区拆成训练状态、当前句反馈、能力评分和课后总结，降低评委理解成本。
- 课后报告卡展示本次总分、完成轮数、主要优点、高频问题、推荐表达和下一次训练建议。
- 评分面板补充每项评分解释，说明五维量化反馈的含义。
- README 增加“题目要求对应实现”表格、MVP 发音评测边界和第三方能力说明。

## 对应题目要求

| 题目要求 | 本 PR 展示优化 |
| --- | --- |
| 实时语音对话 | 页面顶部和左侧链路说明中明确展示实时语音对话能力。 |
| 场景化训练 | 训练状态卡片展示当前场景，场景切换保留面试、点餐、会议。 |
| 发音评测 | 评分解释中说明识别清晰度 / 发音可懂度的 MVP 量化依据。 |
| 语法/表达纠错 | 当前句反馈区展示结构化纠错、推荐表达和中文解释。 |
| 课后总结 | 结束后展示正式报告卡片，数据不足时也基于已有对话兜底展示。 |
| 可量化反馈 | 能力评分区展示五维分数和每项含义。 |
| 纠错时机 | 模式说明和链路说明强调逐句反馈适合纠错，沉浸模式结束后统一总结。 |

## 如何测试

1. 启动后端和前端。
2. 打开 `http://localhost:5173`。
3. 切换面试、点餐、会议场景，确认顶部状态卡同步变化。
4. 切换逐句反馈和沉浸式对话，确认模式说明和右侧反馈来源文案变化。
5. 在逐句反馈模式下录入一轮英文，确认用户消息标注为“语音识别结果”，AI 消息标注为“AI 语音回复”。
6. 点击“获取完整纠错评分”，确认右侧展示结构化纠错和能力评分解释。
7. 点击“结束对话并生成课后报告”，确认课后报告区域显示总分、轮数、优点、问题和建议。
8. 执行 `npm run build`，确认前端构建通过。

## 当前限制和后续优化方向

- Pronunciation Clarity 当前是 MVP 级发音可懂度估算，不是专业音素级发音评测。
- 翻译按钮仍调用后端翻译接口，后续可接入更轻量的机器翻译服务降低成本和延迟。
- 页面已展示实时链路设计，但还可以进一步接入真实耗时埋点，分别统计识别耗时、AI 回复耗时和语音播放状态。
- 课后报告兜底展示基于前端已有记录，真实评分仍以 AI 总结接口返回为准。
