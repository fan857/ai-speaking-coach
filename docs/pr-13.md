# PR 13：新增口语能力量化报告

## 本 PR 做了什么

- 将课后总结评分从四维升级为五维：
  - 流畅度
  - 识别清晰度 / 发音可懂度
  - 语法准确度
  - 表达自然度
  - 互动完成度
- DeepSeek 总结提示词要求返回 `taskCompletion` 和对应 `scoreReasons`。
- 后端 normalizer 和 mock 兜底统一返回五维评分结构。
- 前端课后总结展示五维评分、每项评分理由、评分依据和综合均分。
- 前端使用 `localStorage` 保存最近 5 次课后总结，展示本机训练趋势。
- README 补充可量化训练报告能力说明。

## 为什么这样做

题目要求不仅是“能实时对话”，还要体现口语能力提升的可量化反馈。单纯给一段总结不够直观，评委也很难判断系统是否真的在做口语训练。

本 PR 用明确 rubric 约束大模型评分，并在页面展示评分依据。识别清晰度在当前 MVP 中定义为“ASR 可识别程度 / 发音可懂度”，不冒充专业音素级发音评测。

## 核心实现思路

- `prompts.py` 的总结提示词新增 `taskCompletion`，要求模型根据场景目标和对话推进情况评分。
- `ai_client.py` 对真实模型返回结果做结构归一化，缺失字段时提供保守默认理由。
- `text_utils.py` 的 `build_scores` 支持第五项评分，保持 mock 和真实模型结构一致。
- `mock_feedback.py` 的课后总结兜底增加互动完成度理由。
- `App.jsx` 增加 `SUMMARY_HISTORY_KEY`，把最近 5 次总结均分保存到浏览器本地。
- 总结卡片展示综合均分和相对上次的变化，形成可演示的训练趋势。

## 如何测试

```bash
python -m compileall backend\app
npm run build
git diff --check
```

手动测试：

- 完成一轮对话后点击“结束并总结”，确认课后总结显示五项评分。
- 确认每项评分下方有对应理由。
- 确认评分依据说明包含“对话转写、ASR 可识别程度、语言质量”，且没有宣称音素级发音评分。
- 连续完成两轮总结，确认综合均分区域显示相对上次的趋势变化。
- 断开 DeepSeek Key 后，确认 mock 兜底总结仍包含五维评分。
