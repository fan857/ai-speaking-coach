# PR 9：新增沉浸式全英文对话模式

## 本 PR 做了什么

- 新增“逐句反馈 / 沉浸对话”模式切换。
- 逐句反馈模式保留原有即时纠错、评分和学习建议。
- 沉浸对话模式下，AI 全程只用英文和用户对话。
- 沉浸对话模式下不即时展示纠错、评分和学习建议。
- 后端 `POST /api/practice/coach` 支持 `mode=immersive`。
- DeepSeek 提示词按模式拆分：
  - `feedback`：英文回复 + 中文纠错 + 评分
  - `immersive`：只返回自然英文回复
- mock 兜底也支持沉浸模式，真实 AI 失败时仍可继续英文对话。

## 为什么这样做

真实英语口语陪练不能每句话都打断用户纠错，否则会影响表达节奏。沉浸对话模式先保证用户能连续开口、AI 能及时英文回应，等用户点击“结束对话”后再统一点评。

本 PR 只实现沉浸式对话模式，不生成全程总结。总结属于独立功能，放到下一 PR。

## 核心实现思路

- 前端新增 `practiceMode` 状态，支持 `feedback` 和 `immersive`。
- 切换模式时重置当前对话，避免不同模式的反馈混在一起。
- 前端请求 `/api/practice/coach` 时传入 `mode`。
- 沉浸模式下隐藏即时纠错、学习建议和评分面板。
- 后端 `PracticeRequest` 增加 `mode` 字段，默认值为 `feedback`。
- 后端根据 `mode` 选择不同提示词和 mock 兜底逻辑。
- 沉浸模式仍保留 AI 英文朗读和“重读”能力。

## 如何测试

1. 启动后端：

```bash
npm run dev:backend
```

2. 启动前端：

```bash
npm run dev:frontend
```

3. 打开：

```text
http://localhost:5173
```

4. 选择“逐句反馈”模式，输入英文并提交，确认仍显示纠错、评分和学习建议。

5. 切换到“沉浸对话”模式，输入英文并提交，确认：

- AI 回复只使用英文。
- 右侧不展示即时纠错、评分和学习建议。
- 页面显示沉浸对话说明。
- AI 回复会自动朗读。
- “重读”按钮仍可重新播放 AI 回复。

6. 点击“结束对话”，确认页面提示下一 PR 会生成全程点评和训练重点。
